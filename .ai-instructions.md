## ðŸš€ Common Tasks & How to Handle Them

### Task: "Explain this concept"
**Action:** Use the `concept_explainer` template style.
1.  **Analogy**: Start with a non-technical comparison.
2.  **Technical Reality**: Map the analogy to the code/protocol.
3.  **Code**: Show it in action (Python simulation, CAPL script, etc.).

### Task: "Document this bug fix"
**Action:** Use the `troubleshooting_story` template style.
1.  **The Symptom**: What went wrong? (Make it dramatic).
2.  **The Hunt**: How did we find it?
3.  **The Fix**: The actual code change.
4.  **The Lesson**: What did we learn?

### Task: "Create a new guide"
**Action:** Use the `learning_doc` template style.
1.  **Goal**: What will the user be able to do by the end?
2.  **Prerequisites**: What do they need?
3.  **Steps**: Clear, numbered steps with code.
4.  **Verification**: How do they know it worked?
